Obejct(对象）

JavaScript 是一种基于原型而不是基于类的语言。基于类的面向对象语言，比如 Java 和 C++，是构建在两个不同实体之上的：类和实例。

基于原型的语言（如 JavaScript）并不存在这种区别：它只有对象。基于原型的语言具有所谓原型对象(prototypical object)的概念。

用Object.creat() 方法创建对象。它允许你为创建的对象选择一个原型对象，而不用定义构造函数

变量作用域

ECMAScript 6 之前的 JavaScript 没有语句块作用域（一般来说就是一对大括号），现在let是块级作用域，var是函数级作用域。

函数内部可以直接读取全局变量，函数外部无法读取函数内的局部变量

var和let两种声明全局变量的方式，产生的结果不同

变量提升

JavaScript 变量的另一个不同寻常的地方是，你可以先使用变量稍后再声明变量而不会引发异常。这一概念称为变量提升。在 ECMAScript 6 中，let（const）同样会被提升变量到代码块的顶部但是不会被赋予初始值

参数

默认参数

在JavaScript中，函数参数的默认值是undefined。意思是当函数需要有参数时，如果没有参数传递给函数，那么参数值默认为undefined。

剩余参数

当参数数量不确定时，参数可以表示为数组。

参数传递

原始参数（比如一个具体的数字）被作为形参传递给函数，值不会改变。传递一个对象作为参数，而函数改变了这个对象的属性，这样的改变对函数外部是可见的

arguments对象

函数的实际参数会被保存在一个类似数组的arguments对象中。

箭头函数

预定义函数

函数提升

对于函数来说，只有函数声明会被提升到顶部，而函数表达式不会被提升。

函数提升仅适用于函数声明，而不适用于函数表达式

闭包

出于种种原因，我们有时候需要得到函数内的局部变量。所以我们在函数的内部，再定义一个函数。并把这个函数作为返回值返回，然后拿到这个函数，我们也就拿到了函数内部的局部变量。

所以，闭包就是能够读取其他函数内部变量的函数。

闭包的用途

  1. 前面提到的读取函数内部的变量
  2. 让这些变量值始终保持在内存中

在这段代码中，result实际上就是闭包f2函数。它一共运行了两次，第一次的值是999，第二次的值是1000。这证明了，函数f1中的局部变量n一直保存在内存中，并没有在f1调用后被自动清除。

为什么会这样呢？原因就在于f1是f2的父函数，而f2被赋给了一个全局变量，这导致f2始终在内存中，而f2的存在依赖于f1，因此f1也始终在内存中，不会在调用结束后，被垃圾回收机制（garbage collection）回收。
