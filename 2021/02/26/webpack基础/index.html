开发环境webpack配置

  1. CSS: 提取成单独文件，而不是加入js中（MiniCssExtractPlugin.loader） -> 浏览器兼容性处理（postcss-loader） -> 压缩（OptimizeCssAssetsPlugin）
  2. JS：js代码规范（eslint-loader） -> 浏览器兼容性处理（babel-loader） -> 压缩(mode=’production’)
  3. HTML: 以原来的html文件，生成新的html文件（HtmlWebpackPlugin） -> html中的图片资源加载（（html-loader）-> 压缩（HtmlWebpackPlugin）
  4. 图片：加载（url-loader）
  5. 字体或者其他文件（file-loader）

webpack性能优化

开发环境性能优化

  1. 优化打包构建速度

    HMR（hot module replacement），当修改任意模块，只重新打包修改的模块，不重新打包所有模块。

    样式文件: 依赖style-loader的热加载功能，虽然在生产环境不使用style-loader

    js文件：默认不能使用HMR功能

    html文件：默认不能使用HMR功能，一旦使用HMR，同时会导致问题：html文件不能热更新。由于我们html文件很少，所以html不需要使用HMR

    ​ 解决：html文件不能热更新，修改entry入口，将html文件引入，注意这需要html-loader

  2. 优化代码调试

    source-map: 提供源代码到构建后代码的映射技术（构建后代码出错，可以通过映射追踪源代码的错误），打包会生成**.map**文件
